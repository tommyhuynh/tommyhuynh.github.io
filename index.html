Tom-icles | MEAN.js Application

January 2016

Tom-icles is a small MEAN.js application that displays, in a Reddit style upvote system, articles that Tommy likes. I wanted to use my knowledge on server-side and client-side development I gained from my job to build a whole full-stack application from scratch using MEAN.js, a language I was not familiar with before he start of this project.

I utilized MongoDB and Mongoose to store the article url and upvotes, Express to build a RESTful Node API and Angular AJAX $http calls to build an applicatoin with no refreshes.

During my free time, I aim to deploy this application, as well as fix some bugs that still exist.

TL;DR | Google Chrome Extension

Cal Hacks | October 2015

TL;DR can be describe as "SparkNotes but for web articles!" It summarize any article by just (1) highlighting the text and (2) clicking "TLDR". Adjust the "Summary Size" settings to scale the length of the summaries to your liking. Perfect for those long news/social articles that no one has time to read!

My role in this project consisted of building the actual Google Chrome extension itself. This involved learning JavaScript on the spot and utilizing Content Scripts to interact with the web-page in order to pull the highlighted text into our alogorithm. I also helped develop the "popularity" algorithm that determines which sentences best summarizes the article. A quick description of the algorithm:

  1. Parses the article into words and sentences.
  2. Assigns "popularity" points to each occurance of a word and if that word occurs in the title.
  3. Sentence "popularity" is then formed from the sum of those words' values.
  4. Return a certain percanatage of the most highly ranked sentences in chronological order.
This Google Chrome Extension was built by my friends and I during Cal Hacks 2015. Released October 18, 2015.
